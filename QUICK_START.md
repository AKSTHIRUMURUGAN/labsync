# 🚀 LabSync - Quick Start Guide

## Installation (2 minutes)

```bash
# 1. Install dependencies
npm install

# 2. Run development server
npm run dev

# 3. Open browser
http://localhost:3000
```

## Access Template Editor

```
Login → Faculty Dashboard → Templates → Create Template
```

Or directly: `http://localhost:3000/faculty/templates/create`

---

## Create Your First Template (5 minutes)

### 1. Configure Settings (Left Sidebar)
```
✏️ Template Name: "Ohm's Law Experiment"
📚 Subject: Physics
📊 Difficulty: Beginner
📝 Description: "Verify Ohm's Law..."
```

### 2. Add Sections (Click Buttons)
```
➕ Heading    → "Aim"
➕ Text Block → Enter aim text
➕ Image      → Upload circuit diagram
➕ Divider    → Visual separator
➕ Heading    → "Procedure"
➕ Text Block → Step-by-step instructions
```

### 3. Format Content (Use Toolbar)
```
🔤 Bold, Italic, Underline
📐 Align: Left, Center, Right
🎨 Colors and Highlighting
🔗 Links and Images
📋 Lists (Bullet/Numbered)
```

### 4. Manage Images
```
📤 Drag & drop files
🔗 Or paste image URL
📏 Size: Small/Medium/Large
📍 Align: Left/Center/Right
```

### 5. Reorder Sections
```
👆 Click drag handle (⋮⋮)
🔄 Drag to new position
✅ Drop to place
```

### 6. Preview & Save
```
👁️ Click "Preview" → See student view
✏️ Click "Edit" → Continue editing
💾 Click "Publish Template" → Make live
📄 Or "Save as Draft" → Continue later
```

## Common Workflows

### Physics Experiment
```
1. Heading: "Aim"
2. Text: Objective
3. Heading: "Theory"
4. Text: Explanation
5. Image: Diagram
6. Heading: "Procedure"
7. Text: Steps
8. Image: Setup
```

### Chemistry Lab
```
1. Heading: "Introduction"
2. Text: Background
3. Image: Apparatus
4. Heading: "Safety"
5. Text: Warnings (highlighted)
6. Heading: "Procedure"
7. Text: Steps
```

### Biology Practical
```
1. Heading: "Objective"
2. Text: Goals
3. Image: Microscope
4. Heading: "Procedure"
5. Text: Instructions
6. Heading: "Observations"
7. Text: Template
```

## Tips & Tricks

### ✅ Do's
- ✅ Use clear, descriptive headings
- ✅ Keep text blocks concise
- ✅ Add images for clarity
- ✅ Preview before publishing
- ✅ Save drafts regularly

### ❌ Don'ts
- ❌ Don't overcrowd sections
- ❌ Don't use tiny images
- ❌ Don't skip preview
- ❌ Don't forget to save
- ❌ Don't use too many colors

## Troubleshooting

### Image won't upload?
```
✓ Check file size (<5MB)
✓ Use PNG, JPG, GIF, WebP
✓ Try URL instead
```

### Can't drag sections?
```
✓ Click drag handle (⋮⋮)
✓ Hold and drag
✓ Refresh page if stuck
```

### Formatting not saving?
```
✓ Check internet connection
✓ Try saving again
✓ Check browser console
```

### Preview looks different?
```
✓ Clear browser cache
✓ Check CSS conflicts
✓ Try different browser
```

## Best Practices

1. **Plan Structure First**
   - Outline sections before building
   - Use consistent naming

2. **Format Consistently**
   - Same heading levels
   - Uniform spacing
   - Consistent colors

3. **Optimize Images**
   - Compress before upload
   - Use appropriate sizes
   - Add descriptive captions

4. **Test Thoroughly**
   - Preview multiple times
   - Check on mobile
   - Test with sample data

5. **Save Regularly**
   - Use draft feature
   - Don't lose work
   - Version control
